Seminar: Thiết kế RESTFUL API document với Swagger

Đăng lúc 03/08/2018
laravel swagger seminar

Mở đầu

Hiện nay, các ứng dụng Software-as-a-Service (SaaS) ngày càng bùng nổ. Theo xu hướng đó, các RESTFUL API cũng trở nên phổ biến và phát triển. Điểm mạnh của API trong mô hình SaaS là một nơi điều khiển các logic của hệ thống. Điều đó đồng nghĩa với việc API sẽ là nơi giao tiếp với nhiều nền tảng khác nhau (Web, mobile app, desktop app...) điều đó gây ra các trường hợp như sau:

  • API chịu trách nhiệm giao tiếp với nhiều bên thuộc các nền tảng khác nhau (Web developer, Mobile developer...)
  • Do thuộc nhiều nền tảng nên sẽ có nhiều feedback từ những nghiệp vụ đặc thù từ các bên khác nhau
  • Đòi hỏi sự nghiêm túc và chuyên nghiệp trong khâu viết và đóng góp các tài liệu về API

Swagger là gì?

Swagger là mã nguồn mở cung cấp các công cụ cho các lập trình viên và các bên khác (kể cả những bên phân tích product, khách hàng) dễ dàng tạo ra một tài liệu RESTful API chuyên nghiệp, hiểu quả và quy chuẩn. Swagger cung cấp cho chúng ta một công cụ như sau:

  • Swagger-Editor: Giúp chúng ta có thể thiết kế lên các API mới hoặc cũ dựa vào file config.
  • Swagger-UI: Tự sinh ra tài liệu và giao diện cho API dự vào file config

OpenAPI Specification

OpenAPI Specification là một định dạng mô tả API dành cho RESTful API. Một file OpenAPI cho phép bạn mô tả toàn bộ API. Được viết bởi định dạng YAML hoặc JSON. Trong file sẽ chứa cái thông tin cơ bản về API như host, basePath, version... Riêng với từng API sẽ có những thông tin cơ bản như path, type, params, response... Các file mẫu về OpenAPI

  • Định dạng JSON
  • Định dạng YAML

Swagger UI

Swagger UI là một mã nguồn mở giúp sinh ra giao diện cho tài liệu từ file config dưới chuẩn OpenAPI. Giao diện được hiện ra rõ ràng và tường minh. Dễ dàng đọc hiểu cho cả lập trình viên lẫn người dùng. Sử dụng file config nhưng hoàn toàn tách biệt tác vụ với nhau.

Chúng ta có thể lên trang chủ swagger.io để tham khảo cách sử dụng. Một trong các cách phổ biến nhất là tải thư viện về sau đó mở Swagger UI lên trình duyệt bằng file index.html trong thư mục dist.

Tài liệu về API được viết một cách tường minh và rõ ràng trên giao diện html của Swagger UI. Có thể test các API trực tiếp trong giao diện (tương tự postman).

Cách làm việc với swagger theo hai hướng tiếp cận

Việt viết document cho Swagger có hai cách tiếp cận chính như sau:

  • Top-down approach:Thiết kế các API trước khi code
  • Bottom-up approach:Từ các API có sẵn thiết kế file config để mô tả chúng.

Swagger & Laravel

Laravel là một framework PHP mạnh mẽ trong việc tạo ra các RESTful API cùng cộng đồng vô cùng lớn nên chúng ta có thể thấy rất nhiều mã nguồn mở trợ giúp cho việc tích hợp và làm việc với Swagger cùng Laravel.

L5-Swagger

L5-Swagger là một thư viện hỗ trợ khá tốt cho toàn bộ công đoạn làm việc với Swagger bao gồm hiển thị UI và generate file config từ các API có sẵn. Document của thư viện cũng mô tả khá rõ ràng về các chức năng, và các cài đặt, tích hợp L5-Swagger vào project Laravel.

Điểm mạnh:

  • Dễ dàng sử dụng, setting.
  • Tác vụ phân chia rõ ràng.
  • Phần mô tả trong comment tường minh và tương đối sát với file config.

Điểm yếu:

  • Do phải đặt tả tương đối nhiều trong comment sẽ làm code bị to ra gây khó chịu trong quá trình dev và đọc code
  • Nếu quá nhiều sẽ đặt ra câu hỏi là generate như vậy liệu có hơn nhiều so với tự design file config thuần

Có một hướng đi khác đó là thay vì trong phần comment chúng ta sẽ thiết lập ngay trong phần code. Bài hướng dẫn Painless API documentation in PHP with Swagger

Mở rộng

Không chỉ với Laravel mà các cộng đồng khác cũng được hỗ trợ rất nhiều trong việc làm việc với Swagger. Có rất nhiều thư viện hỗ trợ dành cho các ngôn ngữ khác như Swagger-docs, rswag (Ruby on rails), Express Swagger Generator, Express Comments Swagger(ExpressJS), Django REST Swagger (Django). Nên sẽ không có vấn đề nếu như chúng ta có nhu cầu sử dụng Swagger trong một dự án không dùng Laravel hoặc là partner sử dụng các công nghệ khác

Ngoài ra Swagger còn kha khá các tool vệ tinh phụ trợ xung quanh với sự đóng góp mạnh mẽ từ cộng đồng

Kết

Trong thời đại mà các API đang trở thành xương sống của các ứng dụng đa nền tảng và đòi hỏi sự liên kết của nhiều bên. Swagger và OpenAPI là những công cụ mạnh mẽ đễ giải quyết vấn đề gắn kế các bên, theo dõi thông tin, tiến độ và kiểm tra API một cách dễ dàng.

Dễ dàng sử dụng với Laravel và các công nghệ phổ biến khác (RoR, Express, Django...)

Giúp việc thiết kế và lập trình trở nên tường minh và dễ dàng hơn trong testing kể cả với những người không phải lập trình viên
